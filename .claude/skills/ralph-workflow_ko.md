# Ralph 워크플로우

Ralph Loop 모드를 통한 지속적 작업 실행에 대한 도메인 지식입니다.

## 핵심 철학

### 시지프스 원칙

> "영원히 바위를 굴리도록 저주받은 시지프스처럼, 당신은 할 일 목록에 묶여 있습니다. 멈추지 않습니다. 포기하지 않습니다. 바위는 정상에 도달할 때까지 - 모든 작업이 완료될 때까지 굴러갑니다."

Ralph 모드는 끈질긴 지속성을 구현합니다. 활성화되면 Claude는 완료될 때까지 할 일 목록에 묶입니다.

## Ralph 모드 작동 방식

### 1. 작업 분해
복잡한 작업을 받으면 즉시 분해:

```python
# 예: "사용자 인증 구현"
tasks = [
    "인증 플로우 설계",
    "사용자 모델 생성",
    "비밀번호 해싱 구현",
    "로그인 엔드포인트 생성",
    "로그아웃 엔드포인트 생성",
    "JWT 생성 구현",
    "JWT 검증 미들웨어 구현",
    "단위 테스트 작성",
    "통합 테스트 작성",
    "API 문서 추가",
]
```

### 2. 할 일 목록 생성
작업 시작 전에 포괄적인 할 일 생성:

```
[pending] 인증 플로우 설계
[pending] 사용자 모델 생성
[pending] 비밀번호 해싱 구현
...
```

### 3. 순차적 실행
한 번에 하나의 작업 실행:

```
[completed] 인증 플로우 설계
[in_progress] 사용자 모델 생성  ← 현재
[pending] 비밀번호 해싱 구현
...
```

### 4. 지속적 검증
각 작업 후:
- 작업이 진정으로 완료되었는지 확인
- 해당되면 관련 테스트 실행
- 오류 확인
- 다음 대기 작업으로 이동

## 개발 워크플로우와의 통합

### Ralph + TDD
```
ralph 루프의 각 작업에 대해:
    1. 실패하는 테스트 작성 (RED)
    2. 최소 솔루션 구현 (GREEN)
    3. 리팩토링 (IMPROVE)
    4. 작업 완료 표시
    5. 다음 작업으로 이동
```

### Ralph + Plan
```
1. /plan <기능>        → 상세 계획 수립
2. 사용자가 계획 승인
3. /ralph-loop         → 계획 작업 실행
4. 완료 검증
```

### Ralph + Code Review
```
1. 모든 구현 작업 완료
2. /code-review        → 모든 변경사항 리뷰
3. 발견된 이슈 수정
4. 변경사항 커밋
```

## 실행 패턴

### 패턴 1: 기능 구현
```
/ralph-loop 장바구니 기능 추가

생성된 할 일 목록:
1. [ ] Cart 모델 생성
2. [ ] CartItem 모델 생성
3. [ ] add_to_cart() 구현
4. [ ] remove_from_cart() 구현
5. [ ] get_cart_total() 구현
6. [ ] 단위 테스트 작성
7. [ ] API 엔드포인트 작성
8. [ ] 문서 추가

실행: 작업 1 → 작업 2 → ... → 작업 8
```

### 패턴 2: 버그 수정 체인
```
/ralph-loop 모든 인증 버그 수정

생성된 할 일 목록:
1. [ ] 버그 #123 조사 (토큰 만료)
2. [ ] 토큰 만료 이슈 수정
3. [ ] 회귀 테스트 추가
4. [ ] 버그 #124 조사 (비밀번호 재설정)
5. [ ] 비밀번호 재설정 플로우 수정
6. [ ] 회귀 테스트 추가
7. [ ] 전체 테스트 스위트 실행
8. [ ] 변경 로그 업데이트

실행: 중단 전 체인 완료
```

### 패턴 3: 리팩토링 세션
```
/ralph-loop 데이터베이스 레이어 리팩토링

생성된 할 일 목록:
1. [ ] 현재 구조 분석
2. [ ] 테스트 커버리지 확인
3. [ ] 리포지토리 패턴 추출
4. [ ] User 쿼리 마이그레이션
5. [ ] Order 쿼리 마이그레이션
6. [ ] 각 마이그레이션 후 테스트 실행
7. [ ] 문서 업데이트
8. [ ] 최종 검증

실행: 체계적 리팩토링
```

## 완료 기준

### 시지프스 검증 체크리스트

모든 작업 세션을 종료하기 전 검증:

| 확인 | 질문 |
|------|------|
| 할 일 목록 | pending/in_progress 작업이 없는가? |
| 기능 | 요청된 모든 기능이 작동하는가? |
| 테스트 | 모든 테스트가 통과하는가? |
| 오류 | 해결되지 않은 오류가 없는가? |
| 품질 | 코드가 프로덕션 준비 상태인가? |

**어떤 답변이라도 아니오면 작업을 계속하세요.**

## 차단 상황 처리

### 막혔을 때
```
1. 차단 상황을 명확히 문서화
2. 해결을 위한 하위 작업 생성
3. 할 일 목록에 추가
4. 해결 가능한 부분 계속 진행
5. 나중에 차단 상황으로 돌아옴
```

### 요구사항이 불명확할 때
```
1. 구체적인 명확화 질문
2. 응답 대기
3. 답변에 따라 할 일 목록 업데이트
4. 실행 계속
```

### 테스트 실패 시
```
1. 작업을 완료로 표시하지 않음
2. 실패 디버그
3. 이슈 수정
4. 테스트 다시 실행
5. 통과할 때만 완료로 표시
```

```

```

## 설정

### 자동 활성화 트리거
다음을 감지하면 Ralph 모드 활성화:
- 명시적 `/ralph-loop` 명령
- "끝날 때까지 멈추지 마"
- "반드시 완료해"
- "전부 끝내"
- "끝날 때까지 계속해"

### 수동 비활성화
- `/cancel-ralph` 명령
- "루프 중단"
- "ralph 취소"
- "중단"
