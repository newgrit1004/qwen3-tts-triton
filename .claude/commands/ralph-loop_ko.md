# /ralph-loop 명령어

작업이 완료될 때까지 계속되는 자기 참조 실행 루프를 시작합니다.

## 사용법
```
/ralph-loop <작업 설명>
```

## 이 명령어의 역할

1. **활성화**: 지속적 실행 모드
2. **강제**: 중단 전 작업 완료
3. **유지**: 전체 과정에서 할 일 목록 추적

## Ralph 철학

> "바위는 정상에 도달할 때까지 멈추지 않는다."

시지프스가 영원히 바위를 굴리도록 저주받은 것처럼, Ralph 모드는 Claude를 할 일 목록에 묶습니다. 모든 작업이 완료될 때까지 작업이 계속됩니다.

## 사용 시점

- 복잡한 다단계 구현
- 포기해서는 안 되는 작업
- "끝날 때까지 멈추지 마"라고 말할 때
- 완전한 전달이 필요한 중요 기능

## 자동 활성화 신호

다음을 감지하면 Ralph 모드가 자동으로 활성화됩니다:
- "끝날 때까지 멈추지 마"
- "반드시 완료해"
- "전부 끝내"
- "모든 작업이 끝날 때까지"

## 실행 흐름

```
1. 작업 분석 → 포괄적인 할 일 목록 생성
2. 첫 번째 작업 시작 → in_progress로 표시
3. 작업 완료 → completed로 표시
4. 남은 작업 확인 → 대기 중인 작업 있으면 계속
5. 완료 검증 → 모든 작업 완료? 중단 : 계속
```

## 할 일 목록 관리

### 필수 동작
- 작업 시작 전에 할 일 생성
- 정확히 하나의 작업만 `in_progress`로 표시
- 완료 즉시 `completed`로 표시
- 대기 중인 작업이 있으면 절대 중단하지 않음

### 작업 상태
| 상태 | 의미 |
|------|------|
| `pending` | 아직 시작 안 함 |
| `in_progress` | 현재 작업 중 |
| `completed` | 성공적으로 완료 |

## 검증 체크리스트

모든 ralph-loop 세션을 종료하기 전:

- [ ] **할 일 목록**: pending/in_progress 작업 없음
- [ ] **기능**: 요청된 모든 기능 작동
- [ ] **테스트**: 모든 테스트 통과 (해당되는 경우)
- [ ] **오류**: 해결되지 않은 오류 없음
- [ ] **품질**: 프로덕션 준비 완료

**체크되지 않은 항목이 있으면 작업을 계속하세요.**

## 지속 강제

불완전한 작업으로 중단하려고 하면:

```
[시스템 리마인더 - 할 일 지속]
할 일 목록에 미완료 작업이 남아 있습니다.
다음 대기 중인 작업을 계속 진행하세요.
허락을 구하지 않고 진행하세요.
완료되면 각 작업을 완료로 표시하세요.
모든 작업이 끝날 때까지 멈추지 마세요.
```

## 예제 세션

```
사용자: /ralph-loop JWT로 사용자 인증 구현, 테스트, 문서화 포함

Claude:
전체 구현을 위한 할 일 목록 생성:
1. [ ] 인증 아키텍처 설계
2. [ ] 사용자 모델 및 스키마 생성
3. [ ] JWT 토큰 생성 구현
4. [ ] 토큰 검증 미들웨어 구현
5. [ ] 로그인/로그아웃 엔드포인트 추가
6. [ ] 단위 테스트 작성 (80%+ 커버리지)
7. [ ] 통합 테스트 작성
8. [ ] API 문서 추가

작업 1 시작: 인증 아키텍처 설계...
[8개 작업 모두 완료될 때까지 계속]

최종 검증:
✓ 모든 할 일 완료
✓ 테스트 통과
✓ 오류 없음
✓ 문서화 완료

Ralph 루프가 성공적으로 완료되었습니다.
```

## 취소

활성 ralph 루프를 취소하려면:
```
/cancel-ralph
```

또는 명시적으로:
- "루프 중단"
- "ralph 취소"
- "작업 중단"

## 다른 명령어와의 통합

Ralph 모드는 다음과 함께 작동:
- `/plan` - 먼저 상세 계획 수립
- `/tdd` - 각 작업 내에서 TDD 사용
- `/code-review` - 구현 후 리뷰

## 모범 사례

1. **구체적으로**: 상세한 작업 설명이 도움됨
2. **분해**: 큰 작업 → 작은 하위 작업
3. **루프 신뢰**: 자연스럽게 완료되도록 함
4. **끝에서 검토**: 최종 출력 품질 확인

## 관련 명령어

- `/cancel-ralph` - 현재 루프 중단
- `/plan` - ralph 실행 전 사전 계획
- `/tdd` - ralph 작업 내에서 테스트 주도
