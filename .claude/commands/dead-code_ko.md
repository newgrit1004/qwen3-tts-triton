# /dead-code 명령어

skylos를 사용하여 사용하지 않는 함수, 클래스, 변수를 찾습니다.

## 사용법
```
/dead-code [경로]
```

**인자:**
- `경로`: 분석할 디렉토리 (기본값: `src/`)

## 이 명령어의 역할

1. **실행**: 지정된 경로에서 skylos 정적 분석
2. **식별**: 사용하지 않는 함수, 클래스, 변수
3. **제공**: 각 발견 항목에 대한 신뢰도 점수
4. **선택사항**: 보안 이슈 스캔 (하드코딩된 비밀)

## 신뢰도 수준

| 점수 | 의미 | 권장 조치 |
|------|------|-----------|
| 90-100 | 확실히 미사용 | 삭제 안전 |
| 70-89 | 거의 확실히 미사용 | 간단히 검토 후 삭제 |
| 50-69 | 아마도 미사용 | 신중히 검토 |
| 30-49 | 가능성 있는 미사용 | 거짓 양성 가능성 높음 |
| 0-29 | 불확실 | 건너뛰기, 아마 사용 중 |

## 실행 단계

### 단계 1: 데드 코드 분석 실행
```bash
uv run skylos [경로] --confidence 60 --gate
```

### 단계 2: 보안 스캔 (선택사항)
```bash
uv run skylos [경로] --danger --secrets
```

### 단계 3: 보고서 생성

```markdown
## 데드 코드 분석 보고서

### 요약
- 스캔한 파일: 15개
- 미사용 함수: 5개
- 미사용 클래스: 2개
- 미사용 변수: 8개
- 보안 이슈: 1개

### 높은 신뢰도 (삭제 안전)

| 항목 | 유형 | 파일:줄 | 신뢰도 | 이유 |
|------|------|---------|--------|------|
| old_helper | 함수 | src/utils.py:45 | 98% | 호출 없음 |
| LegacyClass | 클래스 | src/models.py:120 | 95% | 인스턴스화 없음 |
| UNUSED_CONST | 변수 | src/config.py:10 | 92% | 참조 없음 |

### 중간 신뢰도 (검토 필요)

| 항목 | 유형 | 파일:줄 | 신뢰도 | 참고 |
|------|------|---------|--------|------|
| _internal_func | 함수 | src/api.py:80 | 72% | 동적 호출 가능성 |
| temp_data | 변수 | src/main.py:25 | 65% | 테스트에서 사용 확인 |

### 보안 발견 사항

| 심각도 | 파일:줄 | 이슈 |
|--------|---------|------|
| 높음 | src/config.py:15 | 하드코딩된 API 키 패턴 |
```

## 프레임워크 인식 분석

Skylos는 인기 프레임워크의 패턴을 인식합니다:

| 프레임워크 | 자동 제외 패턴 |
|-----------|----------------|
| Django | `models.Model`, 뷰, 마이그레이션 |
| Flask | 라우트 핸들러, 데코레이터 |
| FastAPI | 엔드포인트 함수, 의존성 |
| Pytest | 테스트 함수, 픽스처 |

## 삭제 워크플로우

### 단계 1: 높은 신뢰도 항목 검토
```python
# src/utils.py:45 - 신뢰도: 98%
def old_helper():  # <- 삭제 안전
    pass
```

### 단계 2: 삭제 전 확인
- 동적 호출 확인: `getattr()`, `importlib`
- 테스트 전용 사용 확인
- 플러그인/확장 패턴 확인

### 단계 3: 삭제 및 테스트
```bash
# 삭제 후
make test        # 모든 테스트 통과 필수
make typecheck   # 타입 오류 없음
```

## 보안 스캐닝

### 보안 기능 활성화
```bash
uv run skylos . --danger --secrets
```

### 감지 항목
- 하드코딩된 비밀번호/토큰
- 소스 코드의 API 키
- 개인 키
- 데이터베이스 자격증명

### 보안 보고서 형식
```markdown
### 보안 이슈 발견

#### 높음: 하드코딩된 API 키
**파일**: src/config.py:15
**패턴**: `API_KEY = "sk-..."`
**수정**: 환경 변수로 이동

#### 중간: 가능한 비밀번호
**파일**: src/db.py:30
**패턴**: `password = "admin123"`
**수정**: 시크릿 매니저 사용
```

## 다른 도구와의 통합

1. **데드 코드 먼저 실행**: 복잡도 분석 전 코드베이스 축소
2. **그 다음 복잡도**: 남은 코드에 `/complexity`
3. **마지막으로 테스트**: 회귀 없음 확인을 위해 `make test`

## 출력 형식

```bash
# CI/CD용 JSON 출력
uv run skylos . --output json > dead-code.json

# GitHub 통합용 SARIF
uv run skylos . --output sarif > dead-code.sarif

# 스프레드시트 분석용 CSV
uv run skylos . --output csv > dead-code.csv
```

## 관련 명령어

- `/complexity` - 코드 복잡도 분석
- `/code-quality` - 전체 품질 분석 실행
- `/security-check` - 전체 보안 검토
