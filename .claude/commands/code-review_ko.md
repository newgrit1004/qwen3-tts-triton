# /code-review 명령어

커밋되지 않은 변경사항에 대해 자동화된 코드 리뷰를 수행합니다.

## 사용법
```
/code-review
```

## 이 명령어의 역할

1. **호출**: `code-reviewer` 에이전트
2. **분석**: 모든 커밋되지 않은 변경사항
3. **보고**: 심각도별 이슈

## 리뷰 프로세스

### 단계 1: 변경사항 감지
```bash
git diff          # 스테이징되지 않은 변경사항
git diff --staged # 스테이징된 변경사항
```

### 단계 2: 각 파일 분석
수정된 각 파일에 대해:
- 보안 취약점
- 코드 품질 이슈
- 모범 사례 위반

### 단계 3: 보고서 생성

```markdown
## 코드 리뷰 보고서

### 요약
- 리뷰한 파일: 5개
- 심각 이슈: 0개
- 높음 이슈: 2개
- 중간 이슈: 5개

### 발견 사항

#### [높음] 코드 품질: 함수 너무 김
**파일**: `src/processor.py:45-120`
**이슈**: `process_data` 함수가 75줄
**수정**: 더 작은 함수로 추출

#### [중간] 모범 사례: 타입 힌트 누락
**파일**: `src/utils.py:30`
**이슈**: `helper` 함수에 반환 타입 없음
**수정**: `-> str` 반환 타입 어노테이션 추가
```

## 이슈 카테고리

### 심각 (반드시 수정)
- 하드코딩된 자격증명
- SQL 인젝션
- 경로 순회
- 안전하지 않은 역직렬화

### 높음 (수정 권장)
- 50줄 초과 함수
- 500줄 초과 파일
- 4단계 초과 중첩
- 오류 처리 누락
- 디버그 문

### 중간 (고려)
- 독스트링 누락
- 매직 넘버
- 중복 코드
- 성능 이슈

## 승인 기준

| 상태 | 조건 |
|------|------|
| 승인 | 심각/높음 이슈 없음 |
| 경고 | 중간 이슈만 있음 |
| 거부 | 심각/높음 발견 |

## 자동 수정 명령어

리뷰 후 실행:
```bash
make lint-fix   # 린팅 이슈 자동 수정
make format     # 코드 포맷팅
make typecheck  # 타입 검증
make test       # 테스트 실행
```

## 예제 출력

```
## 코드 리뷰: 승인

### 요약
- 리뷰한 파일: 3개
- 심각: 0개
- 높음: 0개
- 중간: 2개

### 중간 이슈

1. [중간] `src/api.py:45` - 독스트링 추가 고려
2. [중간] `src/utils.py:12` - 매직 넘버 86400

### 권장사항
- 상수 추가: `SECONDS_PER_DAY = 86400`
- `make lint-fix` 실행하여 스타일 이슈 자동 수정
```
