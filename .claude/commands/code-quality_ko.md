# /code-quality 명령어

데드 코드 감지와 복잡도 분석을 결합한 종합적인 코드 품질 분석을 실행합니다.

## 사용법
```
/code-quality [경로]
```

**인자:**
- `경로`: 분석할 디렉토리 (기본값: `src/`)

## 이 명령어의 역할

1. **데드 코드 분석**: 미사용 코드 찾기 (skylos)
2. **복잡도 분석**: 이해하기 어려운 코드 찾기 (complexipy)
3. **보안 스캔**: 하드코딩된 비밀 감지 (선택사항)
4. **통합 보고서**: 우선순위가 매겨진 개선 권장사항

## 실행 단계

### 단계 1: 데드 코드 분석 실행
```bash
uv run skylos [경로] --confidence 60 --gate
```

### 단계 2: 복잡도 분석 실행
```bash
uv run complexipy [경로] --max-complexity-allowed 15
```

### 단계 3: 보안 스캔 (선택사항)
```bash
uv run skylos [경로] --danger --secrets
```

```bash
make dead-code
```

## 임계값 설정

### 환경 변수
```bash
export COMPLEXIPY_MAX_COMPLEXITY=15    # 10=엄격, 15=보통, 20=느슨
export SKYLOS_CONFIDENCE=60            # 0-100 (높을수록 더 확실)
```

### pyproject.toml
```toml
[tool.complexipy]
max-complexity-allowed = 15

[tool.skylos]
complexity = 10
nesting = 4
```

## CI/CD 통합

```yaml
# .github/workflows/quality.yml
- name: 코드 품질 검사
  run: |
    uv run skylos src/ --gate --confidence 80
    uv run complexipy src/ --max-complexity-allowed 15
```

## 관련 명령어

- `/complexity` - 상세 복잡도 분석
- `/dead-code` - 상세 데드 코드 분석
- `/refactor-clean` - 코드 이슈 자동 수정
- `/security-check` - 전체 보안 검토
