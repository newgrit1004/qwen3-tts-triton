# Ralph Loop 규칙

Ralph Loop 실행 모드에 대한 필수 규칙입니다.

## 핵심 규칙

### 규칙 1: 대기 작업이 있으면 절대 중단하지 않음

```
금지:
- 할 일 목록에 대기 항목이 있을 때 중단
- 완료 없이 "좋은 진전"이라고 선언
- 작업이 남아있을 때 "계속할까요?" 질문
```

작업이 남아있으면 자동으로 계속해야 합니다.

### 규칙 2: 한 번에 하나의 In-Progress

```
올바름:
[completed] 작업 A
[in_progress] 작업 B  ← 하나만
[pending] 작업 C
[pending] 작업 D

잘못됨:
[in_progress] 작업 A  ← 여러 in_progress
[in_progress] 작업 B
[pending] 작업 C
```

### 규칙 3: 즉시 완료 표시

```
올바름:
1. 기능 구현 완료
2. 작동 확인
3. 즉시 completed로 표시
4. 다음 작업으로 이동

잘못됨:
1. 기능 구현 완료
2. 다음 작업 시작
3. 이전 작업 완료 표시 잊음
4. 진행 상황 추적 불가
```

### 규칙 4: 시작 전 할 일 생성

```
올바름:
1. 작업 수신: "사용자 인증 추가"
2. 모든 하위 작업으로 할 일 목록 생성
3. 그 후 작업 시작

잘못됨:
1. 작업 수신: "사용자 인증 추가"
2. 즉시 코딩 시작
3. 진행 상황 추적 없음
```

### 규칙 5: 완료 전 검증

작업을 완료로 표시하기 전:
- [ ] 코드가 오류 없이 컴파일/실행
- [ ] 관련 테스트 통과 (해당되는 경우)
- [ ] 명백한 버그 도입 없음
- [ ] 기능/수정이 실제로 작동

## 지속 강제

### 시스템 리마인더 트리거

미완료 작업으로 중단하려고 하면:

```
[시스템 리마인더 - 할 일 지속]
할 일 목록에 미완료 작업이 남아 있습니다.
다음 대기 중인 작업을 계속 진행하세요.
허락을 구하지 않고 진행하세요.
완료되면 각 작업을 완료로 표시하세요.
모든 작업이 끝날 때까지 멈추지 마세요.
```

### 필수 응답

이 리마인더를 보면:
1. 리마인더 인정
2. 다음 대기 작업 식별
3. in_progress로 표시
4. 작업 계속
5. 허락을 구하지 않음

## 최종 검증 체크리스트

ralph-loop 완료 선언 전:

| 확인 | 검증됨? |
|------|---------|
| 모든 할 일 `completed` 표시 | [ ] |
| `pending` 작업 없음 | [ ] |
| `in_progress` 작업 없음 | [ ] |
| 모든 테스트 통과 | [ ] |
| 처리되지 않은 오류 없음 | [ ] |
| 기능 작동 | [ ] |

**체크되지 않은 항목이 있으면 중단하지 마세요.**

## 예외 처리

### 합법적인 중단 시점

Ralph 루프가 중단할 수 있는 경우:
- 사용자가 명시적으로 `/cancel-ralph` 말함
- 사용자가 "중단", "취소" 말함
- 외부 의존성 차단 (예: API 키 대기)
- 명확화가 필요한 모호한 요구사항

### 차단 상황 처리

```
1. 차단 상황 문서화
2. 구체적인 질문 (필요시)
3. 해결을 위한 하위 작업 생성
4. 가능하면 다른 작업 계속
5. 해결 후 차단된 작업으로 복귀
```

## 통합 규칙

### TDD와 함께
```
ralph 루프의 각 작업에 대해:
- 테스트 먼저 작성 (해당되는 경우)
- 솔루션 구현
- 테스트 통과 확인
- 그 후 완료 표시
```

### 코드 리뷰와 함께
```
모든 작업 완료 후:
- /code-review 실행
- 발견된 이슈 수정
- 검증 재실행
- 그 후 커밋
```

### Git과 함께
```
다음 전까지 커밋하지 않음:
- 모든 ralph 작업 완료
- 모든 테스트 통과
- 코드 리뷰 통과 (수행한 경우)
```

## 명령어 참조

| 명령어 | 동작 |
|--------|------|
| `/ralph-loop <작업>` | 지속 실행 시작 |
| `/cancel-ralph` | 실행 중단 |
| "계속" | 일시 중지된 루프 재개 |
| "중단" | 루프 취소 |

## 강제 우선순위

```
1. 최고: 모든 작업 완료
2. 높음: 할 일 목록 정확성 유지
3. 중간: 변경 후 테스트 실행
4. 표준: 코드 품질 규칙 준수
```

바위는 정상에 도달할 때까지 멈추지 않습니다.
